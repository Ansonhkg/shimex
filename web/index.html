Shimex GitHub Readme

A local provider gateway for Codex Desktop.

Shimex sits between Codex Desktop and any LLM provider — Anthropic, DeepSeek, Ollama, LM Studio, Cloudflare Workers AI, ClinePass — behind an OpenAI-compatible HTTP daemon. Your original Codex install is never modified.

View on GitHub → Install

What it does

  • Copies your real /Applications/Codex.app into a managed Shimex.app with an isolated profile and model catalog. The original Codex app stays untouched.
  • Exposes an OpenAI-compatible gateway at http://127.0.0.1:18765/v1: /v1/chat/completions, /v1/responses, /v1/models, and a Codex picker catalog.
  • Normalises Anthropic Messages, OpenAI Responses, OpenAI Chat, and Cursor Composer behind the same inbound shape. Streaming SSE everywhere; non-streaming upstreams are wrapped in a single-chunk SSE so the consumer always sees SSE.
  • Discovers models dynamically with a written cache. Catalog reads don’t require a live network call; refresh is best-effort on startup.
  • Holds capabilities to what each provider can actually do. Text-only models reject image input with a typed error instead of dropping parts silently.
  • Keeps secrets out of source. API keys live in the git-ignored .env and are referenced via auth: { type: env, name: … }.

Install

macOS with Codex Desktop installed at /Applications/Codex.app, Node 20+, npm 10+.

git clone https://github.com/Ansonhkg/shimex.git ~/Projects/shimex
cd ~/Projects/shimex
npm install
cp .env.example .env   # add your API keys; .env is git-ignored
npm start              # prepares Shimex.app, launches the daemon at :18765

Then open the managed Shimex.app and the admin UI at http://127.0.0.1:18765/admin.

Verify

npm run status                                 # PID, log path, health
npm run shimex -- doctor                       # prerequisite check
npm run shimex -- providers list               # configured providers
npm run shimex -- models list                  # discovered models
